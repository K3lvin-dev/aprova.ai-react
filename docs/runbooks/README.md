## Ownership & Escalation

- Each runbook should identify an owner and escalation path.

## Required Runbook Sections

Every runbook must include:

1. **Purpose** — what operation or incident this covers
2. **Prerequisites** — access, tools, environment needed
3. **Steps** — numbered, actionable steps
4. **Verification** — how to confirm success
5. **Rollback** — how to undo if something goes wrong
6. **Escalation** — who to contact if the runbook fails
